## 📖 系统概述

### 系统名称

汽车配件管理系统（Auto Parts Management System）

### 系统定位

面向汽车配件销售企业的综合管理系统，提供配件管理、库存管理、采购管理、销售管理等核心业务功能。

### 设计目标

- **高效性**: 快速响应业务操作，支持高并发访问
- **可靠性**: 数据一致性保证，操作可追溯
- **易用性**: 简洁直观的用户界面，降低学习成本
- **可扩展性**: 模块化设计，便于功能扩展
- **安全性**: 完善的权限控制，数据安全保护

### 分层职责说明

| 层级 | 职责 | 主要组件 |
|------|------|----------|
| **Controller** | 接收 HTTP 请求、参数校验、调用 Service、返回响应 | `@RestController`, `@RequestMapping` |
| **Service** | 业务逻辑处理、事务管理、数据校验 | `@Service`, `@Transactional` |
| **Mapper** | 数据库操作、SQL 执行 | `@Mapper`, MyBatis-Plus |
| **Entity** | 数据模型、表结构映射 | `@TableName`, `@TableId` |
| **DTO/VO** | 数据传输对象、视图对象 | 数据封装、转换 |

### JWT Token 结构

```
Header:
{
  "alg": "HS256",
  "typ": "JWT"
}

Payload:
{
  "userId": 1,
  "username": "admin",
  "roleId": 1,
  "exp": 1705478400,
  "iat": 1705449600
}

Signature:
HMACSHA256(
  base64UrlEncode(header) + "." + base64UrlEncode(payload),
  secret
)
```

## 🔌 接口设计

### RESTful API 规范

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/parts | 获取配件列表 |
| GET | /api/parts/{id} | 获取配件详情 |
| POST | /api/parts | 创建配件 |
| PUT | /api/parts | 更新配件 |
| DELETE | /api/parts/{id} | 删除配件 |

### 统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {
  }
}
```

### 错误码定义

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或 Token 无效 |
| 403 | 无权限访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

## 📊 性能优化

### 前端优化

1. **代码分割**: 路由懒加载
2. **资源压缩**: Gzip 压缩
3. **缓存策略**: 静态资源缓存
4. **按需加载**: 组件异步加载

### 后端优化

1. **数据库索引**: 常用查询字段建立索引
2. **分页查询**: 避免全表扫描
3. **连接池**: HikariCP 数据库连接池
4. **缓存**: Redis 缓存（可选）

### 网络优化

1. **CDN**: 静态资源 CDN 加速
2. **HTTP/2**: 启用 HTTP/2 协议
3. **Keep-Alive**: 长连接复用
